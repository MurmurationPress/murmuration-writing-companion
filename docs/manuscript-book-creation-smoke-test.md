## Reproduction checklist

Use a backed-up real vault and a provisional Book 4 title such as `FEVER`. Record the existing trilogy book paths and metadata before starting.

1. Open the Manuscript navigator and confirm the labelled **Create book** action is visible. Also confirm it remains visible in a vault with no recognised book notes.
2. Choose **Create book**, enter `FEVER`, and inspect the proposed folder, `FEVER.md` filename, missing-folder list, and exact Markdown preview. Confirm the preview contains only `type: book` and `title: "FEVER"` frontmatter.
3. Cancel. Confirm no note or folder was created.
4. Reopen the dialog and confirm creation. Confirm exactly one Markdown book note is created and its bytes match the preview.
5. Confirm no Part, Scene, Story World, editorial, compiler, publishing, or other manuscript note was created.
6. Confirm the existing trilogy book files, order, and metadata are unchanged.
7. Confirm `FEVER` appears in the navigator and becomes selected.
8. Confirm the new note opens and contains no generated heading or body.
9. Open Continuity Review and confirm it follows the same explicit `FEVER` book selection.
10. Restart Obsidian. Confirm the existing manuscript projection recognises `FEVER` from its Markdown and restores coherent book scope.
11. Attempt another book whose title differs only by case or surrounding whitespace. Confirm the recognised-title collision is explained before writing.
12. Attempt the same destination path, including a case-only spelling difference. Confirm the existing file/folder collision is blocked.
13. Propose the same `FEVER.md` basename in another folder with a different non-colliding book title. Confirm basename alone is not rejected.
14. If practical, pause or delay metadata-cache processing during a disposable run. Confirm the verified note is preserved, delayed recognition is reported, and **Open note** remains available.
15. After recording results, manually remove the provisional note if it should not become canon. Remove any explicitly-created empty test folders separately; the plugin does not remove them automatically.

Also exercise a Unicode title and path, punctuation that changes only the suggested filename, a missing nested folder, a Windows-reserved filename, traversal, duplicate `.md`, and a parent segment occupied by a file.
