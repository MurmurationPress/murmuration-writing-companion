# The Murmuration Manifesto

Writing is difficult enough. Your tools shouldn't make it harder so every feature must reduce the author's cognitive load.

We believe:

- Authors should own their manuscripts.
- Plain text is the future.
- Software should reduce cognitive load.
- The manuscript is sacred.
- Writing tools should disappear while writing.
- Small, focused applications are better than monoliths.
- Open formats outlive software.

Everything in this repository exists to support those principles.

## Natural Language First

The project uses the language of authors, not the language of implementation.

Authors write books, parts, chapters, scenes, notes and passes.

They do not work in files, pages, records, entities or configuration objects.

The user interface, documentation, issue titles and code should preserve this ontology wherever practical.

## Code Transparency

The code should reveal the product.

A developer reading the source should see the same concepts the author sees in the interface:

- Chapter
- Chapter Note
- Selected Text Note
- Editorial Pass
- Checklist
- Writing Session
- Manuscript

Implementation details should remain secondary.

When implementation language is unavoidable, it should be contained at the boundary with Obsidian or the filesystem.
